# 历史与收藏功能技术方案

WebBrowser 的第一块**持久化层**:历史(自动记录访问过的页面)与收藏(手动保存网址)。两者此前都是内存态、关窗即失;本次补上 JSON 落盘 + 下拉面板 UI。设计上刻意复用既有下载管理器的模式(DI 单例服务 + `ObservableCollection` + `Popup` 面板),保持架构一致性,**零新依赖**(`System.Text.Json` 是 net8 自带)。

---

## 1. 设计决策

| 决策点 | 选择 | 理由 |
|---|---|---|
| 存储格式 | JSON(`System.Text.Json`,net8 自带) | 零新依赖,符合"精简"定位;几千条记录内性能足够 |
| 历史 UI | 工具栏 🕒 按钮 + 下拉面板 | 复用下载面板的 `Popup` 模式 |
| 收藏 UI | 工具栏 ★ 按钮 + 下拉面板 | 同上;★ 点击打开面板,面板内可收藏当前页 |
| 收藏结构 | 扁平列表(无文件夹) | MVP 最简,数据结构与 UI 复杂度都低 |
| 数据模型 | 纯 POCO(不继承 `ObservableObject`) | 利于 JSON 序列化,避免 MVVM 噪声 |

## 3. 服务层:防抖落盘 + 容错

[Services/HistoryService.cs](../Services/HistoryService.cs) 与 [Services/BookmarksService.cs](../Services/BookmarksService.cs) 是 DI 单例 + `IDisposable`,镜像 [DownloadManagerService](../Services/DownloadManagerService.cs) 的简单模板。核心机制:

- **内存集合**:`ObservableCollection<T>` 供 UI 直接绑定。
- **防抖落盘**:`DispatcherTimer`(1s)。每次集合变更 reset 定时器,导航/收藏操作停止 1s 后才写盘——避免连续导航每页都做 IO。
- **原子写**:`File.WriteAllText(tmp)` → `File.Replace(tmp, dst, null)`(目标不存在则 `File.Move`)。崩溃不会写到一半损坏文件。
- **加载容错**:启动读文件 `try/catch`,`FileNotFoundException` / `JsonException` → 当作空集合。文件损坏不崩。
- **Dispose flush**:[App.xaml.cs](../App.xaml.cs) 的 `OnExit` 调 `Dispose`,强制把残余 dirty 落盘。

历史另有**去重 + 上限**:

- 同 URL 再次访问 → `VisitCount++` + 更新 `LastVisited` + 移到集合顶部(面板"最近在前")。
- 上限 1000 条,超出删尾部,防止 `history.json` 无限增长。

## 4. 历史记录的数据流

记录点 = [WebViewTab.OnNavigationCompleted](../WebView/WebViewTab.cs):

```csharp
private void OnNavigationCompleted(object? sender, CoreWebView2NavigationCompletedEventArgs e)
{
    _vm.IsLoading = false;

    // 只记成功的 http(s) 导航:排除 about:/data:/blob:/javascript: 和失败加载。
    // Core.Source 是 WebView2 规范化后的 URL,作稳定去重 key。
    if (e.IsSuccess
        && Uri.TryCreate(Core.Source, UriKind.Absolute, out var uri)
        && (uri.Scheme == Uri.UriSchemeHttp || uri.Scheme == Uri.UriSchemeHttps))
    {
        var title = string.IsNullOrWhiteSpace(Core.DocumentTitle) ? Core.Source : Core.DocumentTitle;
        _history.Record(Core.Source, title);
    }
}
```

要点:

- 只记**成功的 http(s) 导航**——过滤 `about:` / `data:` / `blob:` / `javascript:` 与失败加载。
- URL 用 `Core.Source`(WebView2 规范化后),作为稳定去重 key。
- title 取 `Core.DocumentTitle`(导航完成时通常已就绪),空则回退 URL。

### 构造穿透:把 HistoryService 注入每个 tab

镜像 `DownloadManagerService` 既有的穿透路径(它是这条路的范本):

```
TabLifecycleService            (构造注入 HistoryService)
  └─ OpenTabAsync: new TabViewModel(_downloadManager, _history)
       └─ new WebViewTab(this, downloadManager, history)
            └─ OnNavigationCompleted 里调 _history.Record(...)
```

[App.xaml.cs](../App.xaml.cs) 把三个服务注册为单例,DI 容器自动解析依赖顺序。

## 5. 收藏与 ★ 状态

### 收藏入口(设计演变)

最初 ★ 按钮 = "收藏当前页",但没有"查看收藏列表"的入口(只能 `Ctrl+Shift+O`),用户找不到收藏在哪。改为:

- **点 ★** → 打开收藏面板(`ToggleBookmarksCommand`);面板顶部有 **"Add this page"** 按钮(当前页已收藏时变 "Remove")直接收藏当前页。
- **`Ctrl+D`** → 快速收藏/取消当前页(不开面板)。
- ★ 图标**实心/空心仍反映**当前页是否已收藏(`IsCurrentPageBookmarked`)。

### ★ 状态跟踪(`IsCurrentPageBookmarked`)

★ 图标与面板按钮都绑 `IsCurrentPageBookmarked`。重算时机有三处([MainViewModel](../ViewModels/MainViewModel.cs)):

```csharp
partial void OnActiveTabChanged(TabViewModel? value)
{
    // 解订旧 tab、订新 tab 的 PropertyChanged(监听 Address)
    if (_bookmarkedTab is not null)
        _bookmarkedTab.PropertyChanged -= OnActiveTabPropertyChanged;
    _bookmarkedTab = value;
    if (value is not null)
        value.PropertyChanged += OnActiveTabPropertyChanged;
    RecomputeBookmarkState();
}

private void OnActiveTabPropertyChanged(object? sender, PropertyChangedEventArgs e)
{
    if (e.PropertyName == nameof(TabViewModel.Address))
        RecomputeBookmarkState();
}

private void RecomputeBookmarkState()
    => IsCurrentPageBookmarked = ActiveTab is not null && _bookmarks.ContainsUrl(ActiveTab.Address);
```

- 切标签 → `OnActiveTabChanged`(CommunityToolkit `[ObservableProperty]` 生成的 partial hook)。
- 同标签内导航 → 订阅当前 tab 的 `PropertyChanged`,响应 `Address` 变化。
- 收藏集变化 → `BookmarksService.Items.CollectionChanged`。

### 双按钮互斥(实心/空心星)

★ 用两个叠加 `ui:Button` + `Visibility` 的 `DataTrigger` 互斥——镜像主题切换按钮写法。WPF-UI 的 `Button.Icon` 不能用 Style setter(技术方案既有坑 #11),必须 inline;实心由 `{ui:SymbolIcon Star24, Filled=True}` 切换。

## 6. UI:面板与主题

### Popup 面板(克隆下载面板)

三个面板(下载 / 历史 / 收藏)同构:`Popup` + `UserControl`。开关由 `[ObservableProperty] bool IsXxxOpen` + `[RelayCommand] ToggleXxx` 驱动,`PlacementTarget` 指向工具栏对应按钮。

行内命令(打开 / 删除)走 `RelativeSource AncestorType=UserControl` → `DataContext`(MainVM),`CommandParameter={Binding}`(行模型)——镜像 [TabStripView](../Views/TabStripView.xaml) 标签项的命令路由。整行 `ui:Button` 打开,内嵌右侧 `Dismiss24` 按钮删除(嵌套 Button 互不干扰)。

### 主题适配(关键修复)

最初三个面板背景硬编码 `#FF202020`(深灰),字色继承主题——浅色主题下黑字黑底**不可读**。改为 WPF-UI 主题画刷:

| 元素 | 主题画刷 |
|---|---|
| 面板背景 | `{ui:ThemeResource ApplicationBackgroundBrush}` |
| 边框 / 行分隔 | `{ui:ThemeResource ControlElevationBorderBrush}` |
| 文字 | `{ui:ThemeResource TextFillColorPrimaryBrush}`(设在面板根的 `TextElement.Foreground`) |

`ui:ThemeResource` 是 WPF-UI 的主题资源 markup,亮/暗切换时自动刷新——浅色主题白底深字、深色主题深底浅字,都可读。

### 快捷键

| 快捷键 | 功能 |
|---|---|
| `Ctrl+H` | 历史 |
| `Ctrl+D` | 收藏 / 取消当前页 |
| `Ctrl+Shift+O` | 收藏面板 |

⚠️ `KeyBinding` 多修饰符必须用 `Modifiers="Control+Shift"`(**`+`** 不是 `,`)——后者运行时抛 `XamlParseException: Unrecognized ModifierKeys 'CONTROL,SHIFT'`,而 XAML 编译期不校验该字符串,只能启动时发现。

## 7. 关键风险与踩过的坑

1. **ModifierKeys 分隔符**:`Modifiers="Control,Shift"` 运行时崩溃,必须 `"Control+Shift"`。XAML 编译期不校验,只能启动发现。
2. **★ 状态的订阅管理**:切标签要解订旧 tab 的 `PropertyChanged`、订新 tab 的,否则内存泄漏 + 状态错乱。用 `_bookmarkedTab` 字段跟踪当前订阅的 tab。
3. **JSON Unicode 转义**:`System.Text.Json` 默认把非 ASCII(CJK / emoji)转义成 `\uXXXX`。读回仍正确,但文件肉眼不可读。`JavaScriptEncoder.UnsafeRelaxJsonEscaping` 在本运行时投影里不可用(CS0117),暂留默认转义,代码有注释。
4. **Popup 的 DataContext 继承**:Popup 视觉树独立,但逻辑树 DataContext 仍继承(PanelView.DataContext = MainVM),所以面板里 `{Binding Xxx}` 直接绑 MainVM 的属性/命令可用(这也是下载面板一直能工作的原因)。
5. **防抖落盘 + 强杀**:正常关窗走 `OnExit` → `Dispose` flush;但 `taskkill /F` 强杀进程跳过 `OnExit`,残余 dirty 会丢失——靠 1s 防抖 timer 在运行期已 flush 大部分。
6. **`ui:Button.Icon` 必须 inline**:技术方案既有坑 #11,本次 ★ 双按钮同样遵守。

## 10. 已知限制与后续增强

- **JSON 中文转义**:title 存为 `\uXXXX`(见坑 #3),功能正常但文件可读性差。可用 `JavaScriptEncoder.Create(UnicodeRanges.All)` 改善。
- **历史 / 收藏无搜索 / 分组**:扁平列表,无搜索框、无文件夹。需要时可加。
- **favicon 未持久化**:历史 / 收藏条目不带 favicon(沿用项目 favicon 内存缓存策略)。
- **`bookmarks.json` 仅在有内容时写出**:空收藏集不落盘(`Add` 才 `ScheduleFlush`),首次收藏前文件不存在,属正常。
